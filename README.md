# BookDNA™

**Upload your Goodreads reading history. Discover the genetic code of your literary taste.**

BookDNA is a single-file web app that parses your Goodreads CSV export and generates an interactive "DNA map" of your reading life — clustered by genre, sized by volume, and powered by live AI recommendations.

🔗 **Live app:** [dblasing.github.io/bookdna](https://dblasing.github.io/bookdna)  
📦 **Repo:** [github.com/dblasing/bookdna](https://github.com/dblasing/bookdna)

---

## What it does

- Parses your Goodreads library export (`.csv`) entirely in the browser — no server, no upload
- Detects genres from titles, authors, and Goodreads shelf tags across 40+ keyword patterns
- Renders an interactive bubble map — each bubble is a genre cluster
- Bubble **size** reflects how many books you've read in that genre
- Faint **connection lines** link thematically related genres (e.g. Sci-Fi ↔ Technology, Psychology ↔ Philosophy)
- A pulsing **nucleus** at the center anchors the map with radial grid rings
- **Click any bubble** to open a side panel showing your top-rated books in that genre and 5 live AI recommendations
- Recommendations are generated by Claude (Anthropic's AI) based on your actual reading history in that genre
- Click any recommendation to search it directly on Goodreads

## How to use it

### Step 1 — Export your Goodreads library

1. Sign in at [goodreads.com](https://www.goodreads.com)
2. Go to **My Books**
3. Click **Import and Export** in the left sidebar
4. Click **Export Library** and wait for the file to generate
5. Download the `.csv` file to your machine

### Step 2 — Upload to BookDNA

1. Open [dblasing.github.io/bookdna](https://dblasing.github.io/bookdna)
2. Drag and drop your `.csv` onto the upload zone, or click **Choose File**
3. BookDNA analyzes your library and renders the map (a few seconds)

### Step 3 — Explore your map

| Action | What it does |
|--------|-------------|
| Scroll | Zoom in / out |
| Drag | Pan around the map |
| Hover a bubble | See genre name, book count, and avg rating |
| Click a bubble | Open the genre panel |
| Panel → recommendations | 5 AI-picked books you haven't read |
| Click a recommendation | Search it on Goodreads |

## Architecture

BookDNA is intentionally a single HTML file (`index.html`). All styles, logic, visualization, and API calls live in one file with no build tooling.

### File structure

```
bookdna/
├── index.html      # The entire application
└── README.md       # This file
```

### View system

Three views are managed by toggling a `.hidden` CSS class on `position:fixed` full-viewport `<div>` elements. Only the active view is visible and pointer-interactive.

| View ID | Description |
|---------|-------------|
| `#vLanding` | Drop zone and instructions |
| `#vAnalyzing` | Animated progress ring while parsing |
| `#vDNA` | Topbar + canvas map + side panel |

### Layout — how the canvas fills the screen

The DNA view is a fixed-inset flex column. Below the topbar is `.map-wrap`, a flex row containing the `<canvas>` and the side panel. The canvas has `flex:1` with `min-width:0`. The side panel starts at `width:0` and transitions to `360px` when open.

The critical detail: every flex ancestor in the chain carries `min-height:0`. Without this, browsers refuse to let flex children shrink below their intrinsic content height, which causes the canvas to report zero height. Canvas pixel dimensions are set by measuring `getBoundingClientRect()` after the view is visible — never by reading `clientWidth/clientHeight` directly.

### CSV parsing

`parseCSV()` splits on newlines, handles RFC-4180 quoted fields, and maps columns by header name (case-insensitive partial match). This tolerates Goodreads' exact export format without relying on column position.

`detectGenre()` concatenates title + author + shelf tags and scans against the ordered keyword dictionary. First match wins. Specific multi-word phrases (e.g. `"science fiction"`, `"young adult"`) appear before their component words (`"fiction"`, `"adult"`) to prevent false matches.

### Canvas render loop

All coordinates are in **world space** centered at `(0, 0)`. The `ws(wx, wy)` function transforms to screen space:

```js
screenX = (worldX + panX) * zoom + canvasWidth  / 2
screenY = (worldY + panY) * zoom + canvasHeight / 2
```

Pan mutates `panX/panY` on `mousemove` while dragging. Zoom mutates `zoom` on `wheel`. The `mouseup` listener is on `window` (not the canvas) so releasing the mouse outside the canvas always ends the drag.

Bubbles animate in via a lerp on `animScale`: `b.animScale += (1 - b.animScale) * 0.12` each frame. They are staggered with `setTimeout` offsets on build.

Each bubble renders in layers: glow halo → orbit ring → radial gradient fill → border stroke → inner decoration ring → text labels.

### Click vs drag disambiguation

`dragMoved` is set to `true` on the first `mousemove` event where displacement exceeds 2px from the `mousedown` origin. The `click` handler ignores the event if `dragMoved` is true, so panning never accidentally opens the panel.

### Side panel

The panel slides open by transitioning `width` from `0` to `360px` via CSS. `fitCanvas()` is called with a 340ms delay (matching the CSS transition duration) so the canvas resizes after the panel is fully open, not during the animation.

### AI recommendations and genre classification (Anthropic API)

Both AI features require an Anthropic API key entered on the landing page. The key is stored in `sessionStorage` and included as `x-api-key` in every request. Get a key at [platform.anthropic.com/api-keys](https://platform.anthropic.com/api-keys).

**Genre classification** runs once after CSV parsing, using `claude-haiku-4-5-20251001`:

```js
'x-api-key': key,
'anthropic-version': '2023-06-01',
'anthropic-dangerous-direct-browser-access': 'true',
```

All books are sent in a single batch prompt asking the model to return a JSON object mapping book index → genre label. Haiku is used here for speed and cost (~$0.01 for 350 books).

**Recommendations** run per genre click, using `claude-sonnet-4-6` for higher quality results. The prompt sends up to 12 of the user's books in that genre and asks for 5 recommendations as a raw JSON array.

Both parsers defensively extract JSON by finding the first `[` or `{` and last `]` or `}`, tolerating any preamble. Errors surface as readable messages in the UI.

## Genre detection

BookDNA maps 40+ keyword patterns to canonical genre labels. Patterns are evaluated in order — more specific phrases are listed before shorter, more common words.

| Genre | Keywords matched |
|-------|-----------------|
| Sci-Fi & Speculative | science fiction, sci-fi, speculative, dystopia |
| Mystery & Thriller | mystery, thriller, crime, detective, suspense |
| Fantasy & Magic | fantasy, magic, mythology |
| Biography & Memoir | biography, memoir, autobiography |
| History | historical, history, war |
| Business & Finance | business, finance, economics, management, leadership, entrepreneur, startup |
| Self-Help & Growth | self-help, self help, personal development, productivity |
| Psychology | psychology, behavior, cognitive |
| Philosophy | philosophy, ethics |
| Science & Nature | science, nature, biology, physics, climate |
| Romance | romance, love |
| Graphic & Comics | graphic novel, comics, manga |
| Young Adult | young adult |
| Politics & Society | politics, society, journalism |
| Travel & Adventure | travel, adventure |
| Art & Culture | music, design |
| Spirituality | spirituality, religion, mindfulness |
| Technology | technology, programming, software |
| Humor & Satire | humor, satire |
| Horror | horror, gothic |
| Poetry & Essays | poetry, essay, short stories |
| Health & Wellness | health, medicine, fitness |
| Food & Cooking | cooking, food, culinary |
| Literary Fiction | fiction, literary, novel *(catch-all — listed last)* |

Books not matching any keyword are grouped under **General Fiction**.

## Privacy

Your Goodreads data never leaves your browser. The CSV is parsed entirely client-side in JavaScript. The only outbound network request is to the Anthropic API to generate recommendations — it receives your genre label and a list of book titles and authors, nothing else.

## Changelog

### v4.5 — Refresh button for AI recommendations

- **Added:** "↻ New suggestions" button next to the AI recommendations label in the side panel. Click it to get a fresh set of 5 recommendations for the same genre without closing and reopening the panel.
- **Added:** Button disables itself while the API call is in flight to prevent double-firing, then re-enables when results return.
- **Added:** `currentPanelBubble` state variable tracks which bubble is currently open so the refresh function knows which genre to re-query.

### v4.4 — Fix JSON truncation on large genres

- **Fixed:** Genres with many books (e.g. History) were returning "unexpected format" errors because the response JSON was being cut off mid-array. Root cause: `max_tokens` was set to 1024, which wasn't enough room for 5 recommendations plus any preamble.
- **Fixed:** Bumped `max_tokens` from 1024 to 2048 on recommendation calls.
- **Improved:** Exclusion list in the prompt trimmed to titles only (no authors) to reduce prompt size and leave more room for the response.
- **Note:** The JS dedup filter from v4.3 remains the safety net — already-read books still cannot appear even with a shorter exclusion list in the prompt.

### v4.3 — Guaranteed client-side dedup of already-read books

- **Fixed:** Magpie Murders and other already-read books were still occasionally appearing in recommendations even after v4.2. Root cause: with 300+ books in the exclusion list, the model would miss entries buried in a long prompt.
- **Fix approach:** Two-layer defense. The prompt still sends the full read list as a hint to the model. After the API response, JavaScript filters every recommendation against a `Set` of all read titles (lowercase, trimmed) before anything renders. The model is now just a hint — JS does the actual enforcement. An already-read book cannot get through regardless of what the model returns.

### v4.2 — Exclude full reading list from recommendations

- **Fixed:** Recommendations were suggesting books the user had already read (e.g. *The Sanatorium*, *The Magpie Murders*) because the prompt only sent books from the clicked genre, not the full library. The prompt now sends the user's complete reading list as an explicit exclusion list so the model cannot suggest any book already read, regardless of which genre it was filed under.

### v4.1 — Hallucination guard on recommendations

- **Fixed:** Recommendation prompt now explicitly instructs the model to only suggest books it is 100% certain exist and have been published. Invented titles, fake author/title combinations, and uncertain books are explicitly forbidden. The model is directed to stick to well-known, widely reviewed works only.
- **Note:** This significantly reduces but does not fully eliminate hallucinations — LLMs can still occasionally confuse real books. Always verify an unfamiliar recommendation on Goodreads before adding it to your list.

### v4.0 — API key input + AI-powered genre classification

- **Added:** API key input on the landing page — paste your `sk-ant-…` key, hit Save or Enter. Stored in `sessionStorage` only (gone when you close the tab, never sent anywhere except Anthropic). Included as `x-api-key` in every API call.
- **Added:** AI genre classification — when a key is present, a single batch call to `claude-haiku-4-5-20251001` classifies every book by title and author into the correct genre. The model knows what *The Martian*, *Shoe Dog*, and *Sapiens* actually are; keyword matching does not. Cost is roughly $0.01 for a 350-book library.
- **Added:** Keyword fallback — if no API key is entered, the keyword dictionary still runs so the map renders without a key.
- **Fixed:** Recommendations now show a clear "enter your API key" prompt instead of a cryptic `x-api-key header is required` error.
- **Changed:** `colorForGenre()` now looks up color from a central `GENRE_COLORS` map rather than embedding color inside the keyword dict.
- **Changed:** Classification uses Haiku (fast, cheap); recommendations use Sonnet (higher quality).

### v3.0 — Ground-up rewrite: layout, interactions, reliability

This version rewrote the entire application from scratch to fix fundamental bugs in layout, interaction, and error handling.

**Bug fixes:**
- **Canvas zero height** — The canvas was rendering at 0px height because flex ancestors were missing `min-height:0`, preventing the flex chain from properly constraining children. Fixed by adding `min-height:0` at every flex level and measuring canvas size with `getBoundingClientRect()` after the view is visible.
- **Drag never ending** — `mouseup` was only on the canvas element, so releasing the mouse outside the canvas left `dragging = true` permanently. Moved to `window`.
- **Clicks firing after drags** — The `dragMoved` flag wasn't reliably distinguishing a click from the end of a pan. Rewrote with a 2px displacement threshold.
- **Panel resize race condition** — `fitCanvas()` was called immediately when the panel opened, resizing the canvas mid-animation and causing layout jumps. Now deferred 340ms to match the CSS transition.
- **Silent API failures** — Error messages were being swallowed. The panel now shows the actual error text and a link to the hosted version when the API is called from a local file URL.

**Improvements:**
- View switching rewritten around `position:fixed` full-viewport elements instead of `position:absolute` with flexbox — more predictable across browsers
- `mouseup` moved to `window` so releasing mouse outside canvas always ends pan mode
- Bubble hit detection uses a 1.12× radius multiplier for easier clicking
- Tooltip constrained to canvas bounds so it never clips off-screen
- Goodreads search links include author name for more accurate results
- Genre keyword ordering fixed — multi-word phrases now evaluated before their component words

### v2.0 — API fix + visual redesign

- **Fixed:** AI recommendations were silently failing due to two missing required HTTP headers (`anthropic-version` and `anthropic-dangerous-direct-browser-access`)
- **Added:** Animated starfield background, radial gradient bubble fills, pulsing nucleus, text glow on selected bubble
- **Expanded:** Genre keyword dictionary from ~30 to 40+ patterns
