# BookDNA™

**Upload your Goodreads reading history. Discover the genetic code of your literary taste.**

BookDNA is a single-file web app that parses your Goodreads CSV export and generates an interactive "DNA map" of your reading life — clustered by genre, sized by volume, and powered by live AI recommendations.

🔗 **Live app:** [dblasing.github.io/bookdna](https://dblasing.github.io/bookdna)  
📦 **Repo:** [github.com/dblasing/bookdna](https://github.com/dblasing/bookdna)

---

## What it does

- Parses your Goodreads library export (`.csv`) entirely in the browser — no server, no upload
- Detects genres from titles, authors, and Goodreads shelf tags across 40+ keyword patterns
- Renders an interactive bubble map — each bubble is a genre cluster
- Bubble **size** reflects how many books you've read in that genre
- Faint **connection lines** link thematically related genres (e.g. Sci-Fi ↔ Technology, Psychology ↔ Philosophy)
- A pulsing **nucleus** at the center anchors the map
- **Click any bubble** to open a side panel showing your top-rated books in that genre and 5 live AI recommendations
- Recommendations are generated by Claude (Anthropic's AI) based on your actual reading history in that genre
- Click any recommendation to search it directly on Goodreads

## How to use it

### Step 1 — Export your Goodreads library

1. Sign in at [goodreads.com](https://www.goodreads.com)
2. Go to **My Books**
3. Click **Import and Export** in the left sidebar
4. Click **Export Library** and wait for the file to generate
5. Download the `.csv` file to your machine

### Step 2 — Upload to BookDNA

1. Open [dblasing.github.io/bookdna](https://dblasing.github.io/bookdna)
2. Drag and drop your `.csv` onto the upload zone, or click **Choose File**
3. BookDNA analyzes your library and renders the map (a few seconds)

### Step 3 — Explore your map

| Action | What it does |
|--------|-------------|
| Scroll | Zoom in / out |
| Drag | Pan around the map |
| Hover a bubble | See genre name, book count, and avg rating |
| Click a bubble | Open the genre panel |
| Panel → recommendations | 5 AI-picked books you haven't read |
| Click a recommendation | Search it on Goodreads |

## Running locally

```bash
git clone https://github.com/dblasing/bookdna.git
cd bookdna
open index.html
```

> The AI recommendations require the two Anthropic browser-access headers (see Architecture below). These work on the hosted GitHub Pages version. Running via `open index.html` (file://) may block the API call depending on your browser's CORS policy. To run locally with recommendations working, serve it over HTTP:
> ```bash
> python3 -m http.server 8080
> # then open http://localhost:8080
> ```

## Architecture

BookDNA is intentionally a single HTML file (`index.html`). Everything — styles, logic, visualization, API calls — lives in one file with no build tooling required.

### File structure

```
bookdna/
├── index.html      # The entire application
└── README.md       # This file
```

### Pages / views

The app has three views managed by toggling a CSS class (`off`) on absolute-positioned `<div>` elements:

| View | ID | Description |
|------|----|-------------|
| Landing | `#landing` | Drop zone and instructions |
| Analyzing | `#analyzing` | Progress ring while parsing |
| DNA Map | `#dnaView` | Canvas + side panel |

### CSV parsing

`parseCSV()` splits on newlines, parses each row respecting quoted fields, and maps columns by header name (case-insensitive, partial match). This means it handles Goodreads' exact export format without relying on column position.

`detectGenre()` scores each book by scanning a concatenation of title + author + shelf tags against a 40+ keyword dictionary. First match wins and returns a canonical genre label and hex color.

### Canvas visualization

The map renders on an HTML5 `<canvas>` element via a `requestAnimationFrame` loop. All coordinates are in **world space** (centered at 0,0) and transformed to screen space via:

```
screenX = (worldX + offsetX) * scale + canvasWidth/2
screenY = (worldY + offsetY) * scale + canvasHeight/2
```

Pan is implemented by mutating `offsetX/offsetY` on `mousemove` while dragging. Zoom mutates `scale` on `wheel`. Bubbles animate in using a lerp: `bubble.sc += (bubble.targetScale - bubble.sc) * 0.1`.

Each bubble draws in layers: outer glow → orbit ring → radial gradient fill → inner decoration ring → text labels.

### AI recommendations (Anthropic API)

`fetchRecs()` calls the Anthropic Messages API directly from the browser. Two headers are required for this to work:

```javascript
'anthropic-version': '2023-06-01',
'anthropic-dangerous-direct-browser-access': 'true'
```

The prompt sends the genre name and up to 12 of the user's books in that genre, and instructs the model to return a raw JSON array with no markdown fencing. The response is parsed defensively — it finds the first `[` and last `]` in the response text and parses only that substring, tolerating any preamble the model might include.

Error handling surfaces the actual API error message in the panel UI rather than silently failing.

## Privacy

Your Goodreads data never leaves your browser. The CSV is parsed entirely client-side in JavaScript. The only outbound network request is to the Anthropic API to generate recommendations — it receives your genre label and a list of book titles and authors, nothing else from your library.
